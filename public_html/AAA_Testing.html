AAA TESTING

AAA Testing is basically the full form of Arrange, Act and Assert .

The AAA pattern is a pattern for structuring tests. It breaks each test down into three parts - Arrange, Act, and Assert - where each part is a step leading to the next. The arrange step sets up the test’s input values. The act step prompts the primary function being tested. And finally, the assert step verifies that the output of the function is what was expected.

Arrange : The block of code that sets up the conditions for the test action.

Act : The line of code where the Action is taken on the SUT.

Assert : The block of code that performs the assertions on the state of the SUT after the action.

Example:

{def test_reverse():

"""

list.reverse inverts the order of items in a list, in place

"""

greek = ['alpha', 'beta', 'gamma', 'delta']

result = greek.reverse()

assert result is None

assert greek == ['delta', 'gamma', 'beta', 'alpha'] }

-The list is arranged in the example above.

-The reverse is the action raised in the code above.

-The list is asserted again after declaring the result.

Importance of Testing

- Software testing is a method of determining whether the actual software product meets the expected requirements and ensuring that the software product is free of defects. It entails running software/system components through their paces using manual or automated tools to evaluate one or more properties of interest.

- Software testing is important because if there are any bugs or errors in the software, they can be identified early and fixed before the software product is delivered. A properly tested software product ensures dependability, security, and high performance, which leads to time savings, cost effectiveness, and customer satisfaction.
